

Assignment 3:

Strategic architecture development: In this phase of the project, students identify and document bounded contexts, create a context map of the system, and prototype the architecture. This phase contains three parts:

    Tactical design (according to DD): Divide up the identified bounded contexts in the team (each team member being responsible for one bounded context), refine and model each bounded context using DDD’s basic building blocks (using UML as proposed in https://dl.acm.org/doi/10.1007/s42979-021-00825-z
    Further explore and enrich your domains by performing event storming and domain storytelling. 
    Implementation of the architecture: Implement your design based on the selected architecture (Example: each team member is responsible for one microservice), integrate the microservices with service discovery, centralised configuration, resilience (using patterns like circuit breakers), and a rudimentary user interface for testing the system.

Strategic Architecture Development (Group) – 30 Marks

The architecture development phase is divided into the following three parts: 

Part A Tactical Design (DDD + UML) – 10 Marks

In this part, you need to divide the bounded contexts among team members, refine and model each bounded context using DDD’s building blocks (Entities, Value Objects, Aggregates, Repositories, Domain Services, Domain Events) and UML, as described in the referenced approach. Read Part 1 and Part 9 of Learning Domain Driven Design, UML Diagrams and Lecture Week 7.
